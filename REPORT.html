Toast Feedback System — Universal Pattern

Every component now uses a dual-toast system: red error toasts (7s auto-dismiss) and green success toasts (5s auto-dismiss) with per-operation specific messages.

// Error toast — catches all write errors
useEffect(() => {
  const err = writeError1 ?? writeError2 ?? writeError3;
  if (!err) return;
  const msg = err?.shortMessage ?? err?.message ?? 'Transaction failed';
  setTxError(msg.length > 120 ? msg.slice(0,120) + '…' : msg);
  const t = setTimeout(() => setTxError(null), 7000);
  return () => clearTimeout(t);
}, [writeError1, writeError2, writeError3]);

// Success toast — per-operation specific message
useEffect(() => {
  if (op1Success) { setTxSuccess('Specific message for operation 1'); }
  else if (op2Success) { setTxSuccess('Specific message for operation 2'); }
  else return;
  const t = setTimeout(() => setTxSuccess(null), 5000);
  return () => clearTimeout(t);
}, [op1Success, op2Success]);

🚀 Startup Guide

Full Local Dev Setup

# 1. Ensure Docker Desktop is running

# 2. Start Anvil (if not already running)
docker compose up -d anvil

# 3. Deploy all 35 contracts
docker compose up --force-recreate deployer

# 4. Copy the output NEXT_PUBLIC_ lines into frontend/.env.local
#    (or use the auto-updated .env.local from this session)

# 5. Kill any existing node process and start fresh dev server
taskkill //F //IM node.exe
cd frontend
npm run dev

# 6. Open browser
http://localhost:3000
